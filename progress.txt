        Response: Array of pair objects

    - 优势：

  - **Docker 部署解决方案实现**：
    - 用户需求：使用 Docker 发布项目，包含 3 个功能
      1. 提供 Dashboard 的页面
      2. 每小时定时执行 run_full_pipeline.sh
      3. 复制本地 .env 的最新配置
    - 约束条件：没有 docker-compose，需要用脚本实现
    - 解决方案：完整的 Docker 部署方案
      - Dockerfile：多阶段构建（Node.js + Python）
      - docker-deploy.sh：自动化部署脚本
      - docker-entrypoint.sh：容器启动脚本
      - .dockerignore：优化镜像大小
      - DOCKER.md：完整的部署文档
    - Dockerfile 关键特性：
      - 多阶段构建：分离构建环境和运行环境
      - Alpine Linux：减小镜像体积
      - Cron 任务：每小时执行 run_full_pipeline.sh
      - 健康检查：HTTP GET /health
      - Volume 挂载：.env、data/、logs/
    - docker-deploy.sh 功能：
      - build: 构建 Docker 镜像
      - up: 启动容器（后台运行）
      - down: 停止并删除容器
      - restart: 重启容器
      - ps: 查看容器状态
      - logs: 查看容器日志
      - status: 查看详细状态（健康检查、资源使用）
      - exec: 进入容器执行命令
      - clean: 清理未使用的资源
      - help: 显示帮助信息
    - docker-entrypoint.sh 功能：
      - 创建必要的目录（logs/）
      - 设置 cron 任务（每小时执行一次）
      - 重定向 cron 输出到日志文件
      - 启动 cron 守护进程
      - 启动 Dashboard 服务器
    - Volume 挂载策略：
      - ./.env:/app/.env:ro（只读挂载，支持配置热更新）
      - ./data/raw:/app/data/raw（原始数据文件）
      - ./data/models:/app/data/models（训练好的模型）
      - ./data/predictions:/app/data/predictions（预测结果）
      - ./logs:/app/logs（日志文件）
    - 优势：
      1. 不需要 docker-compose，只使用 Docker
      2. 支持配置热更新（修改 .env 后重启容器即可）
      3. 数据持久化（通过 Volume 挂载）
      4. 定时任务自动化（每小时更新预测）
    - 测试验证：
      - Dockerfile 构建成功
      - docker-deploy.sh 脚本功能正常

      - cron 任务正常执行
