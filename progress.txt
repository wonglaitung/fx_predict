  - **数据去重功能实现**：
    - 问题：布林带（BB_Upper/BB_Middle/BB_Lower）、ADX等技术指标显示为0
    - 根本原因：数据文件包含重复的日期行（最后10行都是2026-03-20，价格相同）
    - 分析过程：
      1. 检查数据文件：发现338行数据，最后10行日期相同
      2. 手动计算布林带：当最后20天价格相同时，标准差为0，导致布林带 collapsed
      3. 确认问题：重复日期导致技术指标计算异常
    - 解决方案：在 `excel_loader.py` 的 `_preprocess()` 方法中添加自动去重
    - 实现代码：
      ```python
      # 自动去重：删除重复的日期行，每个日期只保留第一行
      if 'Date' in df.columns:
          before_count = len(df)
          df = df.drop_duplicates(subset=['Date'], keep='first')
          after_count = len(df)
          if before_count > after_count:
              self.logger.warning(f"发现并删除了 {before_count - after_count} 条重复日期数据")
              self.logger.info(f"去重后数据量: {after_count} 条记录")
      ```
    - 数据量变化：EUR 从338行减少到283行（删除55条重复数据）
    - 处理流程：先排序 → 再去重 → 确保保留正确的那一行
    - 验证结果：
      - 重新运行 pipeline 成功（24秒）
      - 布林带显示正确数值（不再全是0）
      - ADX 显示正常数值
      - 其他技术指标正常计算
    - Git 提交：e95f7b3 - "feat: add automatic deduplication for date rows in data loader"

    - 用户问题："如果数据文件排序不一定对，目前系统读完数据有按日期排序吗"
    - 回答：是的，系统已经自动按日期排序
    - 实现位置：`excel_loader.py` 的 `_preprocess()` 方法（第97-98行）
    - 实现代码：
      ```python
      # 按日期升序排序（旧数据在前）
      if 'Date' in df.columns:
          df = df.sort_values('Date', ascending=True).reset_index(drop=True)
      ```
    - 处理流程：
      1. 按日期升序排序（旧数据在前）
      2. 自动去重（删除重复的日期行，保留第一行）
      3. 生成缺失的 OHLC 列
      4. 前向填充缺失值
    - 重要顺序：先排序 → 再去重，确保 `drop_duplicates(subset=['Date'], keep='first')` 保留正确的那一行
    - 结论：数据文件不需要预先排序，系统会自动处理

  - **经验教训**：
    1. **数据质量的重要性**：重复数据会导致技术指标计算错误，必须在数据加载时清理
    2. **自动去重的必要性**：用户不应该手动编辑数据文件，系统应该自动处理数据质量问题
    3. **数据顺序的处理**：先排序再去重，确保保留正确的数据行
    4. **API 数据结构的完整性**：不同的端点需要不同的数据结构，要根据使用场景设计
    5. **UI 可读性的重要性**：关键信息（如日期）应该显眼显示，提高用户体验
    6. **CSS 样式的层次感**：使用背景色、边框、字体粗细等创建视觉层次
    7. **用户反馈的价值**：用户的问题往往揭示了需要改进的地方
    8. **验证的重要性**：每个修复都需要验证，确保问题真正解决
